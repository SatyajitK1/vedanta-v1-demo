Step 0: Reframe What You’re Designing

You are designing:

A financial control system where users
→ request money
→ system validates
→ approvals happen
→ decisions are tracked
→ insights are generated

So your Figma should reflect:

Decision flow
Governance
Data visibility
Trust & auditability
 Step 1: Identify Core User Personas

Design everything around who uses ACT

 1. Requestor (Employee / Manager)
Raises spend request
Needs quick approval
 2. Approver (Manager / CFO)
Reviews requests
Needs decision clarity
 3. Finance Controller
Monitors budgets
Ensures compliance
 4. Leadership
Needs dashboards & insights

 In Figma, you’ll design flows for each persona separately

 Step 3: Map End-to-End Flow (CRITICAL BEFORE FIGMA)

Before designing screens, draw this (even on paper):

User → Raise Request → ACT Validation → Approval → SAP Execution → Reporting

Break into steps:

User fills request
System checks:
Budget
Rules
Request goes to approver
Approver decides
System triggers SAP
Data flows to dashboard

 Each step = 1–2 Figma screens

 Step 6: Design Principles (VERY IMPORTANT)
1. Clarity > Beauty

Finance tools must be:

Clean
Structured
Trustworthy
2. Show Context for Decisions

Never show just:

“Approve?”

Instead show:

Budget impact
Risk
History
3. Make System Feel Intelligent

Use:

Recommendations
Alerts
Scenario buttons
4. Always Show Status
Pending
Approved
Over budget
At risk
 Step 7: Use Real Data (Don’t Fake It)

Use realistic examples like:

₹12,50,000 request
Budget remaining: ₹10,00,000
Over budget: ₹2,50,000

This makes your design feel real and senior-level
